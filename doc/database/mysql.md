# Mysql
## 执行流程
1. 连接器：建立连接，权限校验
2. 解析器：词法分析，语法分析
3. 优化器：确定执行计划
4. 执行器：执行

## 锁
1. 按互斥程度：共享锁（读锁），排他锁（写锁）
2. 按锁的粒度：全局锁、表级锁、行级锁

## 日志
1. redoLog：物理日志，用于数据库崩溃回复，保证数据的持久性
2. undoLog：逻辑日志，用于事务回滚（保证原子性）和实现mvcc
3. binLog：用于数据主从同步

## 事务
1. 四大特性：ACID
2. 并发事务的问题：脏读、不可重复读、幻读
3. 隔离级别：读未提交、读已提交、可重复读、串行化
4. InnoDB实现可重复读：MVCC+ReadView
